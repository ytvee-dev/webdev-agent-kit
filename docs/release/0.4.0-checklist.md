# WebDev Agent Kit 0.4.0 Release Checklist

Planned tag: `v0.4.0`. Preparing or completing this checklist does not authorize
merging the release pull request, creating the tag, or publishing a GitHub
Release.

## Before Merge

- Confirm milestones 0 through 10 are present in the release pull request and no
  unrelated source changes were added.
- Require successful `skill-pack-ci`, `quality-ci`, and `link-check` results on
  the final head commit.
- Confirm planning-integrity, target-parity, and cross-client smoke gates pass.
- Confirm the runtime inventory remains 19 skills with no Spec Kit dependency,
  CLI, package, MCP server, testing infrastructure, or `.specify/` directory.
- Confirm README was not edited and current user requests remain above durable
  goal and plan artifacts.
- Review `CHANGELOG.md` Unreleased entries against the actual pull request.

## Local Release Candidate

Run from a clean source checkout:

```text
python scripts/validate_skill_pack.py
python scripts/validate_planning_integrity.py
python scripts/build_skill_targets.py
python scripts/validate_planning_target_parity.py
python scripts/validate_planning_smoke.py
python scripts/validate_release_tag.py --tag v0.4.0
python scripts/validate_install_guides.py
python scripts/build_release_archives.py --output release-artifacts --version v0.4.0
python scripts/validate_release_archive.py --directory release-artifacts --version v0.4.0
python scripts/build_release_notes.py --tag v0.4.0 --output release-artifacts/RELEASE_NOTES.md --require-content
```

Build the archives twice in separate empty directories and compare each archive
plus `SHA256SUMS`. Deterministic builds must be byte-identical.

## Cross-Client Inspection

- Codex and VS Code Codex contain the planning rules, templates, and four owning
  skills under `.agents/` with Codex metadata only.
- Claude Code and VS Code Claude contain equivalent planning contracts under the
  native `webdev-agent-kit/` plugin root with exactly one Claude runtime prelude.
- Cursor contains equivalent planning contracts under `.agents/` and the root
  `.cursor/rules/webdev-agent-kit.mdc` entrypoint.
- All six targets pass the eight planning smoke cases: lightweight bypass,
  clarification, coverage, analysis, convergence, precedence, review, and
  resume.
- Stable archives are byte-identical to versioned archives and every checksum is
  present exactly once.
- No archive contains README, CHANGELOG, project state, examples, dependencies,
  caches, unsafe paths, or root project instructions.

## Publish Gate

Only after explicit release approval and merge:

1. Create the exact annotated or lightweight tag `v0.4.0` from the approved
   release commit.
2. Push that tag once. The tag-triggered workflow revalidates source, generated
   targets, planning parity, smoke cases, archives, notes, and tag equality.
3. Confirm all ten stable/versioned archives and `SHA256SUMS` are attached to
   the GitHub Release.
4. Download one archive per canonical target, verify its checksum, extract it in
   an empty fixture, and confirm the documented entrypoint.

Do not repoint the tag or overwrite release assets without a separately approved
recovery plan.
