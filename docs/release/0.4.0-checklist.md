# WebDev Agent Kit 0.4.0 Release Checklist

[← All documentation](../README.md) · [Runtime target contracts](../architecture/runtime-target-contracts.md)

Planned tag: `v0.4.0`. Preparing or running this checklist does not authorize merging the
release pull request, creating the tag, or publishing the GitHub Release.

## Before Merge

- Confirm that stages 0 through 10 are present in the release pull request and that no unrelated source changes were added.
- Require passing `skill-pack-ci`, `quality-ci`, and `link-check` results for the final head commit.
- Confirm that the planning-integrity, target-parity, and cross-client smoke checks passed.
- Confirm that the runtime inventory still contains 19 skills without a Spec Kit, CLI, package, MCP server, test infrastructure, or `.specify/` dependency.
- Confirm that README was not changed and that current user requests still outrank persistent goal and plan artifacts.
- Reconcile the Unreleased entries in `CHANGELOG.md` with the actual pull request.

## Local Release Candidate

Run from a clean source checkout:

```text
python scripts/validate_skill_pack.py
python scripts/validate_planning_integrity.py
python scripts/build_skill_targets.py
python scripts/validate_planning_target_parity.py
python scripts/validate_planning_smoke.py
python scripts/validate_release_tag.py --tag v0.4.0
python scripts/validate_install_guides.py
python scripts/build_release_archives.py --output release-artifacts --version v0.4.0
python scripts/validate_release_archive.py --directory release-artifacts --version v0.4.0
python scripts/build_release_notes.py --tag v0.4.0 --output release-artifacts/RELEASE_NOTES.md --require-content
```

Build the archives twice into separate empty directories and compare each archive together
with `SHA256SUMS`. Deterministic builds must be byte-for-byte identical.

## Cross-Client Inspection

- Codex and VS Code Codex contain planning rules, templates, and four owning skills in `.agents/` with Codex-only metadata.
- Claude Code and VS Code Claude contain equivalent planning contracts at the native `webdev-agent-kit/` plugin root, include the local marketplace manifest, and have exactly one Claude runtime prelude.
- Cursor contains equivalent planning contracts in `.agents/` and the root `.cursor/rules/webdev-agent-kit.mdc` entrypoint.
- All six targets pass the eight planning smoke cases: lightweight bypass, clarification, coverage, analysis, convergence, precedence, review, and resume.
- Stable archives are byte-for-byte identical to the versioned archives, and every checksum appears exactly once.
- No archive contains README, CHANGELOG, project state, examples, dependencies, caches, unsafe paths, or root project instructions.

## Publication Gate

Only after explicit release approval and merge:

1. Create the exact annotated or lightweight tag `v0.4.0` from the approved release commit.
2. Push that tag once. The tag-triggered workflow revalidates sources, generated targets, planning parity, smoke cases, archives, notes, and tag consistency.
3. Confirm that all ten stable and versioned archives plus `SHA256SUMS` are attached to the GitHub Release.
4. Download one archive for each canonical target, verify its checksum, unpack it into an empty fixture, and confirm the documented entrypoint.

Do not retarget the tag or overwrite release assets without a separate approved recovery plan.
